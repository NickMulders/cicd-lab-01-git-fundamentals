01
What this course is about
A CI/CD course for Ignition: not a pure Git course, not a pure CI/CD course.
  • We go deep into Git and CI/CD, but always applied to Ignition, and always made very practical for that setting.
  • Practical tools for your day-to-day work. The goal is what you can use on Monday, not theory for its own sake.
  • Git first. Every CI/CD pipeline triggers off, and reasons about, Git history. If this foundation is shaky, nothing downstream is trustworthy.

03
Get your own copy — fork the repo
Each lab lives in a repo on GitHub. Fork it to your own account first — that gives you a personal copy to commit to, branch, and break things in safely.
STEP 01
Open the repo
Go to the lab's GitHub page — we'll drop the link in the chat.
STEP 02
Click Fork
Top-right of the page. Pick your own account as the owner and confirm.
STEP 03
Clone it
Copy your fork's URL and clone it to your machine, or open it in VS Code.
STEP 04
Work freely
Commit, branch and push — it's all in your fork, separate from everyone else.
Clone your fork

git clone https://github.com/<your-username>/cicd-lab-01-git-fundamentals.git
Because it's your fork, you can push to it freely — your work never collides with anyone else's.
